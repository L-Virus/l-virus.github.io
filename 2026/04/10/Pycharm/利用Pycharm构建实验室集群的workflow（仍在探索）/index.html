利用Pycharm构建实验室集群的workflow（仍在探索）

背景介绍

这是笔者的学校集群，因为组里用服务器的人即将增多，为此写下这篇教程，一来，帮助组里同门免去没必要的探索时间，加快科研效率；二来，做好事，多积德，希望每天增加的不止体重还有实验的指标。

本次教程主要是进行搭建集群到Pycharm的 SFTP 快速文件传输、开机子

因为这个集群跟网上那种云服务器不一样，集群在命令行窗口进行操作是不用申请的，可是也没有 GPU，所以这两个是分开的，但是可以在命令行里做一些跑模型的前置准备工作，比如下载配置环境等，不在此赘述。

前提准备

  1. 被拉入团队的账号
  2. Pycharm 专业版
  3. 有网🤭

一、配置 SFTP

集群端

点击进入命令行窗口

点击右上角的钥匙🔑

能够查看到，个人被配分到的主机地址，端口号，用户名，以及密钥

Pycharm端

进入Pycharm，点击左上角，进入工具，点击部署，选择配置

在类型中选择SFTP

再点击 SSH 配置右边的 ··· ，进入 SSH 配置

将在命令行钥匙处得到的主机地址，端口号，用户名，依次填入，并选择身份验证类型为密钥对，接下来下载私钥文件，在私钥文件中选择下载好的私钥文件（12小时过期），点击测试连接，显示如下图即可

返回部署页面，点击自动检测，检测一下根目录

检测完成后就可以点击映射，将本地项目路径和，集群的部署路径做映射了

全部完成后，可以在右侧和下侧看到内容

二、开机子

看机子

回到集群主页，相信大家没搞过的肯定是一头雾水，先往下翻

上面这幅图是对集群的配置描述，首先左下角的可用资源显示的是集群的资源配置，但是已用这边好像都是0

右下脚存储资源是团队的共享目录和个人主目录的硬盘是用情况

上面可访问队列是具体的算力资源

一般来说大家跑深度模型，用到gpu_4090 虽然是阉割版的4090D，但是也足够了，点击展开可以看到使用情况，就如以下为例，CPU 核数以及 GPU 卡数，是否在使用

开机子

点击 Jupyter 进入开机子的页面

如上图所示，需要关注几个点

  1. 机子名称可以任意取，主要是用于个人区分每个机子在跑的程序
  2. 加速器类型，选择GPU
  3. 镜像选择，下拉选择pytorch的镜像，这个版本没关系，进入jupterlab后可以激活自己在命令行安装的虚拟环境
  4. 资源选择，正常我们跑单卡，选择第一个就行

创建好后，就能进入了

最后，切记，任务结束关机子，不要浪费资源！！！

三、workflow

这是基于目前本人水平所写，如有大佬请联系指正

经过上两步的操作，我们已经打通了本地 Pycharm 和集群的命令行窗口，但是，但是，在命令行窗口并没有显卡（即使你开了机子，也显示不了）

因此目前我探索到的 workflow 就是

  1. 每天上线第一步，先登录集群，下载私钥，这个私钥是12小时过期，每12小时要重新下载
  2. 打开 Pycharm，进入 SFTP 的配置页面，重新加载下载好的私钥文件
  3. 连接成功后，就直接在本地编码，本地编码，本地编码，千万别进 Jupyterlab 中进行编码，不如两个版本容易混乱，同步不来
  4. 当需要命令行运行模型时，再回到 Jupyterlab，中激活环境，切换目录，运行模型
  5. 但有同步问题时，Pycharm 可以通过对比文件来进行手动更正、同步
  6. 目前的同步只能单向同步，也就是说，只能通过修改本地文件来同步集群文件，这就会导致当集群项目文件发生变动时，本地是没有变化的，所以只能通过本地访问集群文件来进行同步

从集群同步本地

比如此时，集群因为跑模型多了些权重文件，我需要本地同步

可以通过集群的文档管理处进行下载，但是切换比较麻烦

我们可以右键所需的文件或文件夹，进行同步，但是需要注意，能进行这步操作的前提是之前映射的文件路径是当前操作目录

也可以通过对比文件，七七八八的功能来防止两地的文件不匹配问题
